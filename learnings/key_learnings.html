Here are the key things I learned about forms and how to create them









A form always has an "action" that's either <get> or <post>.
You create the form -- fieldset -- legend. The contents of forms are labels and inputs! The for should match the id of the input. The "name" is how it is identified at the server level. The required attribute forces the user to input something before submitting the form. The autofocus attribute will have the cursor blinking in this input. The type password doesn't allow others to see the input. The type tel and pattern attribute are used for telephone numbers.There are different input types like decade, select and options like a DROPDOWN INPUT. You can use optgroups for differentiating in the select menu.

The input text type with list=someID with a label -> input and a datalist lets the user type a word and filter the list if it is an exhaustive list. Also learned about radio and checkbox form inputs that have an input first and a label later. I've showcased Radio inputs.
